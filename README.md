
# راهنمای راه‌اندازی و استقرار وب‌سایت با Firebase و Git

این راهنما شما را در فرآیند راه‌اندازی پایگاه داده `Firebase Firestore` و مدیریت کد پروژه با استفاده از `Git` یاری می‌کند.

## ۱. راه‌اندازی پروژه Firebase

برای ذخیره‌سازی اطلاعات کاربران، ما از `Firebase`، پلتفرم توسعه اپلیکیشن گوگل، استفاده می‌کنیم.

### قدم اول: ساخت پروژه در Firebase

1.  به [کنسول Firebase](https://console.firebase.google.com/) بروید.
2.  روی **"Add project"** کلیک کنید.
3.  یک نام برای پروژه خود انتخاب کنید (مثلاً `my-web-app-db`).
4.  مراحل ساخت پروژه را دنبال کنید. می‌توانید Google Analytics را در این مرحله فعال یا غیرفعال کنید.
5.  پس از چند لحظه، پروژه شما آماده خواهد شد.

### قدم دوم: راه‌اندازی Firestore Database

`Firestore` یک پایگاه داده NoSQL و انعطاف‌پذیر برای ذخیره و همگام‌سازی داده‌ها است.

1.  در منوی سمت چپ کنسول Firebase، روی **"Build"** کلیک کرده و سپس **"Firestore Database"** را انتخاب کنید.
2.  روی دکمه **"Create database"** کلیک کنید.
3.  در پنجره باز شده، **"Start in production mode"** را انتخاب کنید و روی **"Next"** کلیک کنید. این حالت امنیت بیشتری را فراهم می‌کند.
4.  مکان سرورهای `Firestore` را انتخاب کنید. بهتر است مکانی را انتخاب کنید که به کاربران شما نزدیک‌تر است. سپس روی **"Enable"** کلیک کنید.

### قدم سوم: ایجاد مجموعه (Collection) برای کاربران

داده‌ها در `Firestore` در مجموعه‌ها (Collections) و اسناد (Documents) سازماندهی می‌شوند. ما یک مجموعه به نام `users` برای ذخیره اطلاعات کاربران ایجاد می‌کنیم.

1.  در صفحه `Firestore Database`، روی **"+ Start collection"** کلیک کنید.
2.  در فیلد `Collection ID`، عبارت `users` را وارد کنید.
3.  روی **"Next"** کلیک کنید. `Firestore` از شما می‌خواهد که اولین سند را ایجاد کنید. می‌توانید یک سند آزمایشی با `Auto-ID` بسازید و فیلدهایی مانند `email` و `name` به آن اضافه کنید.

پایگاه داده شما اکنون آماده است تا اطلاعات کاربران را ذخیره کند. در کد واقعی اپلیکیشن، باید اطلاعات اتصال به Firebase را اضافه کنید تا اپلیکیشن بتواند با این پایگاه داده ارتباط برقرار کند.

## ۲. مدیریت پروژه با Git

`Git` یک سیستم کنترل نسخه است که به شما امکان می‌دهد تغییرات کد خود را ردیابی کرده و با دیگران همکاری کنید.

### قدم اول: نصب Git

اگر `Git` را روی سیستم خود نصب ندارید، از [وب‌سایت رسمی Git](https://git-scm.com/downloads) آن را دانلود و نصب کنید.

### قدم دوم: ایجاد یک مخزن (Repository) محلی

1.  یک ترمینال یا خط فرمان باز کنید.
2.  با استفاده از دستور `cd` به پوشه اصلی پروژه خود بروید.
3.  دستور زیر را برای مقداردهی اولیه یک مخزن `Git` اجرا کنید:

    ```bash
    git init
    ```

### قدم سوم: افزودن و ثبت تغییرات (Add & Commit)

1.  تمام فایل‌های پروژه را به `Git` اضافه کنید تا برای ثبت آماده شوند:

    ```bash
    git add .
    ```

2.  اولین نسخه از کد خود را با یک پیام معنادار ثبت (commit) کنید:

    ```bash
    git commit -m "Initial commit: Project setup"
    ```

### قدم چهارم: اتصال به یک مخزن راه دور (Remote Repository)

برای به اشتراک‌گذاری و پشتیبان‌گیری از کد خود، باید آن را به یک سرویس میزبانی مانند `GitHub`، `GitLab` یا `Bitbucket` ارسال کنید.

1.  در یکی از این سرویس‌ها یک مخزن جدید و خالی ایجاد کنید.
2.  سرویس به شما یک URL برای مخزن راه دور می‌دهد. آن را با استفاده از دستور زیر به پروژه محلی خود متصل کنید (URL را جایگزین کنید):

    ```bash
    git remote add origin https://github.com/your-username/your-repository-name.git
    ```

### قدم پنجم: ارسال کد (Push)

اکنون می‌توانید کد ثبت شده خود را به مخزن راه دور ارسال کنید:

```bash
git push -u origin master
# یا اگر شاخه اصلی شما main نام دارد:
# git push -u origin main
```

از این پس، برای ثبت تغییرات جدید، کافی است مراحل `add`، `commit` و `push` را تکرار کنید:

```bash
# فایل‌های تغییر یافته را اضافه کنید
git add .

# تغییرات را ثبت کنید
git commit -m "Add new feature: User login"

# به سرور ارسال کنید
git push
```
